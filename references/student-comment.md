# 学生评语初稿

## 任务重点

**具体、平衡、不标签化**。不能只写「表现优秀」「有待提高」；不能把一次表现上升为固定判断；**只根据教师提供的记录表达，不编造表现**。

## 执行前读取

- `01-长期资料库/输出模板/学生评语模板.md`
- `01-长期资料库/过往样例/评语样例.md`
- `01-长期资料库/学校要求/评语表达规范.md`
- `02-临时输入/评语素材/` 中的**已脱敏**表现记录

## 执行步骤

1. **核对输入**：记录中是否有具体事例？若过于空泛，提示教师补充观察再生成
2. **逐条映射**：每条评语要点应对应记录中的事实
3. **生成初稿**：积极、具体、平衡；优点与建议并重
4. **标注需核对事实点**：列出每条表述对应的记录来源
5. **输出**并附复核清单

## 输出格式

```markdown
# [学生代号] 期末评语初稿

[正文段落：主要表现 + 具体事例 + 成长建议，一气呵成]

---
## 教师复核清单
- [ ] 每条表述是否有记录依据
- [ ] 是否存在标签化或绝对化判断
- [ ] 语气是否适合该学生和家庭
- [ ] 是否需要调整某条优缺点的权重

## 需核对的事实点
- 「…」→ 依据：记录第X条
```

## 禁止事项

- 批量生成时跳过个别学生的材料核对
- 编造未在记录中出现的职务、奖项、表现
- 使用模板化空话（阳光开朗、积极向上等套话堆砌）
- 把阶段性问题写成 permanent 性格缺陷

## 批量处理

若需全班评语：逐个学生读取独立记录文件，**禁止**用同一模板填空；每个学生输出独立初稿和复核清单。
