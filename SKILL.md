# 班主任 AI 工作台

班主任用 AI，不是让 AI 替你做教育判断，而是把重复的文字整理和初稿生成变得更**稳定、合适、可复用**。

## 核心原则

1. **AI 只生成初稿和整理建议，最终判断由教师负责**
2. **不编造**学校/记录中没有的信息
3. **不暴露**学生隐私（姓名、成绩、家庭、健康等需脱敏）
4. **不对学生标签化**（避免「总是」「从不」「太差」等绝对化表述）
5. **每份对外输出末尾必须附「教师复核清单」**

## 四类任务路由

| 任务 | 重点 | 详细流程 |
|------|------|----------|
| 家长通知改写 | 准确传达，保留原文关键信息 | [references/parent-notice.md](references/parent-notice.md) |
| 班级周总结 | 结构整理，取舍公开信息 | [references/weekly-summary.md](references/weekly-summary.md) |
| 学生评语初稿 | 具体平衡，基于事实不标签化 | [references/student-comment.md](references/student-comment.md) |
| 家校沟通草稿 | 拿捏语气，事实+态度+边界 | [references/home-school-communication.md](references/home-school-communication.md) |

**简单记忆**：通知=准确 | 总结=结构 | 评语=具体 | 沟通=语气

## 项目结构（首次搭建时创建）

在用户指定目录（默认项目根下的 `班主任AI工作台/`）创建：

```
班主任AI工作台/
├── 00-项目规则.md          ← 复制 references/project-rules.md
├── 01-长期资料库/
│   ├── 学校要求/           ← 家校沟通规范、评语表达规范
│   ├── 班级背景/           ← 班级整体说明
│   ├── 输出模板/           ← 四类模板（见 references/templates/）
│   ├── 表达边界/           ← 复制 references/expression-boundaries.md
│   └── 过往样例/           ← 优秀通知、周报、评语样例
├── 02-临时输入/
│   ├── 本周记录/
│   ├── 学校通知/
│   └── 家校沟通素材/       ← 须脱敏
├── 03-Skills/              ← 可选：与本 skill 子流程对应
├── 04-任务记录/
└── 05-输出结果/
```

**执行任何任务前**：先检查 `00-项目规则.md` 和 `01-长期资料库/` 是否存在；不存在则询问用户是否初始化，或按上结构创建最小版本。

## 标准执行流程

```
1. 识别任务类型（通知 / 周总结 / 评语 / 家校沟通）
2. 读取 00-项目规则.md + 对应模板 + 表达边界
3. 读取 02-临时输入/ 中的本次材料
4. 按对应 references/*.md 流程生成初稿
5. 输出到 05-输出结果/ 并记录到 04-任务记录/
6. 末尾附教师复核清单 + 表达风险提醒（如有）
```

## 任务间串联

同一事实可在不同任务间流转（信息分级，非复制粘贴）：

| 原始信息 | 周总结 | 家校沟通 | 评语 | 班级记录 |
|----------|--------|----------|------|----------|
| 共性问题（如忘带材料） | 面向全班温和提醒 | — | — | 教师内部依据 |
| 个别学生反复问题 | — | 脱敏后私信草稿 | 改善后可作成长点 | 保留事实记录 |
| 学校活动安排 | 下周安排段落 | 可提取为通知 | — | — |

**串联流程**：本周记录 → 整理分类 → 周总结初稿 → 提取需配合事项 → 通知/沟通草稿 → 沉淀样例 → 期末评语素材

## 必须人工把关（不可外包）

- 学生事实是否准确
- 表达语气是否适合该家长/情境
- 隐私是否脱敏妥当
- 学校口径是否与正式通知一致
- 教育判断（原因分析、下一步支持策略）

## 快速启动（最小可行）

用户说「帮我做班主任工作流」或首次使用时，引导：

1. 收集一份本周班级记录 → 放入 `02-临时输入/本周记录/`
2. 复制 [references/templates/](references/templates/) 到 `01-长期资料库/输出模板/`
3. 复制 [references/expression-boundaries.md](references/expression-boundaries.md) 到资料库
4. 先做**班级周总结**一项，跑通后再扩展通知、评语、沟通

## 调用示例

**周总结**：
```
参考 01-长期资料库/输出模板/班级周总结模板.md
处理 02-临时输入/本周记录/第N周班级记录.md
要求：家长版初稿、不点名、提醒式表达、附复核清单
```

**评语**：
```
参考 学生评语模板 + 评语表达规范 + 评语样例
处理 02-临时输入/评语素材/学生X阶段性表现记录.md（已脱敏）
要求：只根据记录事实、避免标签化、附需核对事实点
```

**家校沟通**：
```
参考 家校沟通规范 + 表达边界 + 沟通草稿模板
处理 02-临时输入/家校沟通素材/xxx.md
要求：先整理事实、语气平和、附不宜直接发送的风险项
```
